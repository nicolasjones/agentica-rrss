# Agencia de Contenido Autónoma: Metodología OpenSpec

Bienvenido al núcleo de diseño de **Agenmatica**. OpenSpec no es solo documentación, es el **Frenado Técnico** necesario para construir una IA compleja sin romper la arquitectura.

## 📁 Estructura del Nodo de Diseño
- `system_spec/`: Contiene la verdad técnica central del proyecto (Arquitectura global, Modelos base).
- `changes/`: Contiene subcarpetas con el historial de cambios propuestos e implementados.
- `index.md`: El registro de todos los cambios activos en el sistema.

## 🚀 Cómo usar OpenSpec
1.  **Propuesta (`proposal.md`)**: El usuario y el Arquitecto definen el problema y el valor estratégico.
2.  **Arquitectura (`spec.md`)**: Se definen los requerimientos funcionales técnicos.
3.  **Diseño (`design.md`)**: Se especifica la API (FastAPI) y la DB (SQLAlchemy/PostgreSQL).
4.  **Tareas (`tasks.md`)**: Se desglosa en pasos pequeños para que el Ingeniero de Implementación (Claude) los ejecute.

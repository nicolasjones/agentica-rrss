# Proposal: UI Reboot v2.4 (Master Refactor)

## 🎯 Visión
Este cambio propone la reconstrucción estratégica de la capa de presentación (Frontend) de Agenmatica. Tras múltiples iteraciones (Fase 1-6), el código frontend ha acumulado fragmentación. Este refactor consolida las mejores versiones de cada componente en una base de código estable, atómica y acorde al diseño "Electric Circuitry of Chaos".

## 🛠️ Justificación
- **Estabilidad**: Eliminar errores de importación y contextos huérfanos.
- **Experiencia de Usuario (UX)**: Implementar el Planner V5 (Calendarios Grandes) y el Perfil ADN (Reactor de 3 Solapas) de forma nativa.
- **Escalabilidad**: Preparar la interfaz para la omnipresencia del Agente Estratega (Fase 6).

## 📊 Impacto
- **Frontend**: 90% del código será regenerado siguiendo las Specs.
- **Backend**: 0% de impacto (se mantienen los contratos de API existentes).
- **Diseño**: Unificación de tokens CSS y componentes de UI.
